利用hexo+github搭建个人博客（一）

一直苦恼没有一个合适的地方分享自己的想法，在某一天突然想搭建一个个人博客，在网上搜了搜觉得hexo是一个不错的选择，于是有了这一篇文章。该文章写于2020.9.29，仅供参考学习，所有操作是在windows10系统中进行，请读者核对相关细节，希望可以帮助到更多的人。

1.准备工作

主要是hexo的依赖环境和创建github仓库

搭建hexo环境

阅读能力较强的同学可以直接跳转hexo官方文档进行搭建，hexo官方链接：https://hexo.bootcss.com/。

介绍

hexo是一个快速、简洁且高效的博客框架，支持 Markdown，方便部署，并且有很强大的插件可用，帮助你丰富博客。

node.js安装

hexo基于node.js，在安装hexo之前要确保自己的电脑安装了node.js，本次安装是在windows10中进行的，需要在官网中下载windows的node.js，进行手动安装。node.js下载链接：https://nodejs.org/en/。

git安装

git是一个开源的分布式版本控制系统，用于项目控制管理。安装它主要是利用git将搭建好的网站推到 github上，然后通过github提供的github pages静态网页的免费站点进行访问。本次安装是在windows10中进行的，需要在官网中下载windows的git，进行手动安装。git下载链接：https://git-scm.com/。

相关教程很多，官方也有中文安装说明，不再赘述，可以在官网，菜鸟教程，CSDN和博客园等平台进行学习。

创建SSH，一直回车就可以了

会生成两个文件id_rsa和id_rsa.pub，windeows下默认路径为C:\Users\用户名\.ssh

id_rsa为本机密匙，不要轻易泄露

id_rsa.pub为公用密匙，可以给别人，别人通过ssh与主机建立链接

点击github中的setting，点击SSH and GPG keys，点击new SSH key，将id_rsa.pubub里面的内容复制进去，创建。至此，你的电脑就与github建立了SSH链接，可以通过部署命令上传文件。

进入该文件夹，有些主要文件夹和文件要说明

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章（主要在_posts文件夹中）或者一些补充文件
  • themes：主题
  • _config.yml: 博客的配置文件
